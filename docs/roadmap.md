# Engineering Lab Roadmap

## Mission

Build a complete engineering lab while learning how modern computer systems work.

The objective is not to complete courses, but to understand how every component of a real system interacts with the others.

---

# Phase 0 — Foundation ✅

- Repository setup
- GitHub configuration
- Git workflow
- Documentation structure
- English as the working language

---

# Phase 1 — Linux Fundamentals

Goal:
Build a solid understanding of Linux as the operating system that powers most servers.

Topics:
- Installation
- Terminal
- File system
- Permissions
- Users and groups
- Processes
- Services
- SSH

Project:
Configure a Linux virtual machine as the main server for the lab.

---

# Phase 2 — Networking

Goal:
Understand how computers communicate.

Topics:
- IP addressing
- DNS
- DHCP
- Routing
- TCP vs UDP
- HTTP / HTTPS
- Firewalls
- Network troubleshooting

Project:
Configure networking for the Linux server and diagnose common connectivity issues.

---

# Phase 3 — Backend Services

Goal:
Deploy a real application on the server.

Topics:
- APIs
- Web servers
- Reverse proxies
- Databases

Project:
Deploy a backend application that can be accessed from the network.

---

# Phase 4 — Docker

Goal:
Containerize the application.

Topics:
- Images
- Containers
- Volumes
- Networks
- Docker Compose

Project:
Run the entire application stack using Docker.

---

# Phase 5 — Cybersecurity

Goal:
Secure the system.

Topics:
- Authentication
- SSH keys
- Firewalls
- Permissions
- Logs
- Vulnerability basics

Project:
Harden the Linux server following security best practices.

---

# Phase 6 — Monitoring

Goal:
Understand what's happening inside the system.

Topics:
- Logs
- Resource monitoring
- Metrics
- Alerts

Project:
Monitor the server and identify performance issues.

---

# Phase 7 — Cloud

Goal:
Move the lab to the cloud.

Topics:
- Virtual machines
- Deployment
- DNS
- HTTPS
- Backups

Project:
Deploy the complete lab to a cloud provider.

---

# Phase 8 — Automation

Goal:
Reduce repetitive work.

Topics:
- Bash scripting
- Scheduled tasks
- Automation

Project:
Automate common administration tasks.

---

# Final Goal

By the end of this journey I want to:

- Feel comfortable working in Linux.
- Understand how networks operate.
- Deploy complete applications.
- Use Docker confidently.
- Apply cybersecurity fundamentals.
- Manage cloud infrastructure.
- Build systems instead of isolated programs.
- Communicate technical ideas in English.
- Create a professional GitHub portfolio that reflects my growth as an engineer.
